Importing Catalogs in Odoo


You can import Data like Products, Categories, Attributes, Attribute Sets as well as features from Magento 2 to Odoo.

The module makes the import process quick and easy. You can import data manually or via cron scheduler. The data is stored in the form of feeds in Odoo and then automatically creates data on the basis of the settings.

Mapping is created in the module to avoid duplicity of the data.

Exporting Catalogs in Odoo


Odoo Multichannel Magento 2 Connector has made the export process really convenient.

Now, you can export products along with their category from Odoo to ebay.

You can export products directly from the specific product form.

Importing Orders in Odoo


You can import Orders along with the data associated with it from Magento 2 to Odoo.

The import function is simplified with the help of module Odoo Multichannel Magento 2 Connector.

Order statuses (like cancel, delivery slip, and invoice) are automatically imported at the same time. Moreover, data like customers and their addresses, taxes, delivery methods, payment methods, currency is also imported along with it.

Key Features

Below is the detailed list of features for Multi Channel Magento 2.X Odoo Bridge module

Importing Products in Odoo

  • Products are manually imported in bulk
  • You can also import products via cron scheduler
  • Product details (like category, attributes, attribute sets, attribute values) are imported along with products

Importing Categroies in Odoo

  • You can manually import categories in Odoo or, can also import it via cron scheduler

Importing Customers in Odoo

  • Customers are manually imported in bulk
  • You can also import customers via cron scheduler
  • Customers details (like addresses) are imported along with the Customers

Importing Orders in Odoo

  • Orders are manually imported in bulk
  • You can import orders via cron scheduler
  • Order details (like customers and their addresses, taxes, delivery methods, payment methods, currency) are imported along with the orders
  • Order statuses (like cancel, delivery slip and invoice) are also imported along with the orders

Importing Attribute Sets in Odoo

  • You can manually import Attribute Sets in Odoo or, can also import it via cron scheduler

Export Shipment and Invoice Status

  • You can export Shipment and Invoice Status in real-time or, can also export it via cron scheduler

Manually Export Attributes from Odoo

  • You can export attributes on bulk or, export selected attributes

Manually Export Products from Odoo

  • You can export products on bulk or, export selected products

Manually Export Category from Odoo

  • You can export categories on bulk or, export selected categories

Manually Map Orders Statuses

  • You can manually map order statuses from Odoo end
  • Fill the corresponding Order status of Magento 2 and Odoo in the Order Status table

Real time Synchronization

  • You can sync order statuses, shipment (as delivery slip at Magento 2), invoice, cancel status and inventory from Odoo to Magento 2 in real-time
  • The real-time order sync from Magento 2 to Odoo needs to be added through a separate patch. Avail the free copy from our Support Team

Enhancement Dynamic Dashboard

  • Introduced a dynamic dashboard to track the data of Magento 2 to your Odoo
  • Import/Export the data of any channel directly from the dashboard
  • Well-designed Donut Chart and Line Graph to analyze the data
  • The dashboard also offers clickable tabs and data cards

Managing Synchronization History

  • All the synchronization history is managed in Odoo

Donut Chart to Analyze the data

  • Drop down to select the Data Type (Order and Product)
  • Track the data of connected Magento 2 eCommerce instance from the chart

Line Graph to Analyze the data

  • Drop down to select the Data Type (Order, Product, Category and Customer)
  • Track the data through the Line Graph

Clickable Data Cards for the Details

  • View the imported Product, Category, Order and Customers in the Tabs
  • The tabs are clickable and direct you to the respective mapping, feeds as well as sync history
